# 质量检查清单 (Checklist)

SVG → PPTX 转换的质量保证清单。生成 PPT 后逐项对照。

---

## 🔴 P0 · 必须通过

### 1. 节奏规则（最核心）
- [ ] Hero 页与 Non-hero 页交替出现
- [ ] 无连续 3 页以上 Non-hero（Bento 网格页）
- [ ] 封面和结尾页必须是 Hero 类型
- [ ] 每个章节之间有幕封或数据大字报作为呼吸页

### 2. 字体三级分工
- [ ] 大标题/金句/核心数据使用衬线体 (Georgia / Noto Serif SC)
- [ ] 正文/列表使用非衬线体 ("PingFang SC", "Microsoft YaHei", sans-serif)
- [ ] 数据/页码/标注使用等宽体 ("SF Mono", "Fira Code", monospace)
- [ ] 同一页不混用超过 2 种字体族

### 3. 色彩纪律
- [ ] 无 `#FFFFFF` 做卡片底色（用主题卡片浅色值）
- [ ] 无 `#000000` 做文字色（用主题文字深色值）
- [ ] 每页不超过 2 种彩色（主色 + 辅色）
- [ ] 整份 PPT 只使用一套主题色

### 4. SVG XML 合法性
- [ ] 所有 `&` 必须写成 `&amp;`
- [ ] 所有 `<` 必须写成 `&lt;`
- [ ] viewBox 格式正确：`viewBox="0 0 1280 720"`
- [ ] 所有标签正确闭合

### 5. PPTX 可打开性
- [ ] 输出的 .pptx 可以在 PowerPoint 2016+ 中正常打开
- [ ] 每页 SVG 正确嵌入，矢量渲染
- [ ] 右键"转换为图形"可拆为可编辑形状

---

## 🟡 P1 · 转换质量

### 6. Hero 页留白
- [ ] 封面/幕封页留白 ≥ 50%
- [ ] 数据大字报只有一个核心数字
- [ ] 悬念问题页只有一句问话
- [ ] 大引用页引文 + 出处，无多余内容

### 7. Non-hero 页布局
- [ ] 卡片之间 gap ≥ 12px
- [ ] 卡片内边距 ≥ 16px 左右
- [ ] 文字不溢出卡片边界
- [ ] CPL 截断正确（main标题≤18字，body标题≤15字）

### 8. 元数据
- [ ] 每页四角有等宽体元数据（章节号、页码等）
- [ ] 元数据 opacity 0.3-0.5，不干扰内容
- [ ] 元数据不参与内容区网格

---

## 🟢 P2 · 增强特性

### 9. 视觉层级
- [ ] 主卡标题字号 > 普通卡标题 > 正文（20px > 17px > 13px）
- [ ] 高亮数字使用等宽体，字号 ≥ 24px
- [ ] pill 标签使用等宽体，10px

### 10. 装饰克制
- [ ] 无阴影（SVG 中无 filter 模拟阴影）
- [ ] 渐变只出现在封面/幕封背景
- [ ] 内容页背景为纯色

---

## 🔵 P3 · 细节打磨

### 11. 主题一致性
- [ ] 深色/浅色页面有节奏交替
- [ ] 同一章节内卡片圆角统一
- [ ] 分隔线风格统一

---

## 自检流程

```
1. 运行转换脚本
2. 用 PowerPoint 打开 .pptx
3. 整体节奏检查：
   a. 翻一遍，是否感受到"呼吸感"？
   b. 是否有连续 3 页以上密密麻麻的卡片？
   c. 封面和幕封是否有视觉冲击力？
4. 逐页检查：
   a. 字体分工是否正确
   b. 颜色是否符合纪律
   c. 留白是否充分
5. 对照上面清单逐项勾选
6. 发现问题 → 修改 SVG → 重新转换
```

---

## 常见问题

| 现象 | 原因 | 解决 |
|------|------|------|
| PPT 感觉单调 | 全是 Bento 网格页 | 插入 Hero 呼吸页（幕封/大字报/悬念问题） |
| 字体没层级 | 全用同一字体族 | 按三级分工切换衬线/非衬线/等宽 |
| 页面刺眼 | 卡片用了 #FFFFFF | 改用主题「卡片浅」色值 |
| 看起来像代码文档 | 缺少衬线标题 | 大标题必须用衬线体 |
| 信息太密 | 没有 Hero 页缓冲 | 每隔 2-3 页插入一个 Hero 页 |
| 中文乱码 | SVG 文件编码非 UTF-8 | 保存为 UTF-8 |
