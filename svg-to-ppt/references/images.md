# 图片系统规范

## 核心原则

**图片是第一公民** — 只裁底部，顶部和左右保持完整。图片不是装饰，是信息。

---

## 图片存放与命名

### 目录结构
```
项目目录/
├── slides/
│   ├── slide-01.svg
│   ├── slide-02.svg
│   └── ...
└── images/
    ├── 01-cover.jpg
    ├── 03-figma.png
    └── 05-dashboard.png
```

### 命名规则
| 规则 | 说明 |
|------|------|
| **页号补零** | `01` 而非 `1`，方便按顺序排 |
| **英文语义命名** | `cover` 而非 `fengmian`，AI 引用时更清晰 |
| **照片用 JPG** | 照片色彩丰富，JPG 体积小 |
| **截图用 PNG** | 截图带文字，PNG 保真不糊 |
| **单张 ≥ 1600px 宽** | 大屏投影不模糊 |

### SVG 引用方式
```xml
<!-- 相对路径引用，AI 自动生成 -->
<image href="images/01-cover.jpg" 
       x="0" y="0" width="1280" height="720" 
       preserveAspectRatio="xMidYMin slice"/>
```

AI 只需知道"第3页是Figma截图"，会自动写成 `images/03-figma.png`，用户把同名文件丢进文件夹即可。

### 无损换图
**核心：同名覆盖，SVG 一个字不改。**

1. 准备新图片，使用与旧图**完全相同的文件名**
2. 直接覆盖 `images/` 文件夹中的旧文件
3. 重新运行 `svg2pptx-svgblip.py` 生成 PPTX

---

## 图片裁切规则

| preserveAspectRatio 值 | 效果 | 适用场景 |
|------------------------|------|---------|
| `xMidYMin slice` | 水平居中，顶部对齐，底部裁切 | **默认**，90%的图片场景 |
| `xMidYMid slice` | 居中裁切 | 左右对称场景 |
| ❌ `xMaxYMax` | 底部对齐 | **禁止**，切掉视觉焦点 |

## 哪些页面支持图片

### Hero 页

| 页面类型 | 图片角色 | 图片位置 |
|----------|---------|---------|
| **开场封面** | 背景主图 | 全屏或右半 |
| 章节幕封 | ❌ | — |
| 数据大字报 | ❌ | — |
| 悬念问题 | ❌ | — |
| 大引用 | ❌ | — |

### Non-hero 页（Bento 网格）

| 页面类型 | 图片角色 | 图片位置 |
|----------|---------|---------|
| **主从叙事** | 主卡配图 | 主卡内，占 40-50% |
| **主卡+侧栏** | 侧栏配图 | 侧栏卡内 |
| **双主对比** | 对比图 | 左右各一张 |
| 主卡+三等分 | ❌ | — |
| **节奏律动** | 大卡配图 | 大卡区域 |

### 规则
- **每页最多 1 张图片**（封面可例外：左右分屏）
- 图片不与文字抢视觉焦点，图片辅助叙事

## 没有图片时：占位色块

当用户还没提供图片时，不用留白，用**占位色块 + 图标**：

```xml
<!-- 占位色块：圆角矩形 + 图标提示 -->
<rect x="60" y="76" width="560" height="400" rx="12" fill="#0F2038"/>
<rect x="280" y="210" width="120" height="80" rx="8" fill="#1A2E48"/>
<!-- 图标：山和日（SVG path 简化） -->
<path d="M310 265 L330 240 L350 265 Z" fill="#3A4A65"/>
<circle cx="340" cy="235" r="8" fill="#3A4A65"/>
<text x="340" y="290" font-family="'SF Mono', 'Fira Code', monospace" 
      font-size="11" fill="#5A6A85" text-anchor="middle">01-cover.jpg</text>
```

**占位色块设计规则：**
- 背景色比卡片色深一级（如卡片 `#142240` → 占位 `#0F2038`）
- 内部加简化图标（山/日/方框等，用 path 画，不用 emoji）
- 底部等宽文字标注文件名（如 `01-cover.jpg`）
- 占位色块的大小和位置 = 真实图片的大小和位置

## 图片风格约束

### 与主题色的协调
图片应与主题暗色调融合，避免纯白背景图片与深色主题冲突。

### 禁止
- ❌ 纯白背景的图片（与深色主题冲突）
- ❌ 带水印的图片
- ❌ 过于复杂的图片（干扰信息层级）

---

## PPTX 转换器行为

`svg2pptx-svgblip.py` 在打包时会自动处理图片：

1. 读取 SVG 文件
2. 找到所有 `<image href="images/xxx.jpg">` 相对路径
3. 相对于 SVG 文件所在目录，读取图片文件
4. 将图片转为 base64，替换 href 为 `data:image/png;base64,...`
5. 将修改后的 SVG 嵌入 PPTX

**这意味着**：
- SVG 源文件用相对路径 → 用户可以同名覆盖换图
- PPTX 输出是自包含的 → 图片已嵌入，可独立打开
- 换图后只需重新运行转换器，SVG 不用改
