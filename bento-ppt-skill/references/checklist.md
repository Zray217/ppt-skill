# 质量检查清单

这份清单用于在交付前做最后一轮自检。

## P0：不能犯的错

### 1. 事实问题

- 所有数据、时间、政策、公司信息都来自用户材料或已核实来源
- 没有为了凑页数而虚构案例或数字

### 2. 结构问题

- 每页只有一个中心命题
- 没有把整段文档原文直接搬到一页里
- 目录、章节页、内容页、结尾页有明确区分

### 3. JSON 问题

- `PPT_OUTLINE` 是合法 JSON
- `slide_plan.json` 是合法 JSON
- `render_mode` 只有 `engine` 或 `custom_svg`

### 4. 渲染问题

- 所有 SVG 页面都是 `1280x720`
- 如果用引擎输出，SVG 文件名和页码对应

## P1：结构质量

### 1. 节奏

- 不是所有页都同一种页型
- 8 页以上的 deck 通常应有目录页
- 每个大部分开始处最好有章节页或节奏页

### 2. 内容密度

- 封面页不超过 4 个辅助信息点
- 正文页不超过 5 个核心卡片
- 指标页不同时出现太多高对比焦点

### 3. 章节逻辑

- 章节顺序清楚
- 上位章节能概括下位页面
- 页面之间存在合理过渡，不是拼贴

## P2：Bento 引擎页检查

如果页面走 `engine`，重点检查这些：

- 主卡是否在上方，形成 top focus
- 同页高亮焦点是否控制在 1-2 个
- 卡片尺寸类型是否过多
- 窄卡里是否出现长句
- 扁卡里是否只有一句话

如果 wrapper 产出了 `audit.json`，重点查看：

- `top_focus_ok`
- `contrast_budget_ok`
- `size_variety_ok`
- `nested_pattern_ok`

理想状态下全部为 `true`。

## P3：可编辑 PPT 交付检查

如果最终做成 `.pptx`，额外检查：

- 文字仍是文本框，不是图片
- 图表和形状尽量可编辑
- 页内元素命名和层次清楚
- 没有用整页截图替代可编辑内容，除非用户明确接受

## 交付前最后问自己

- 这套 deck 的核心论点是什么，能用一句话说清吗
- 用户最关心的事实和判断是否已经被放进关键页面
- 如果删掉任意一页，会不会更清晰
- 如果某一页太满，是否应该拆成两页而不是继续塞
