# SVG 渲染参考

这份参考来自你现有的“信息架构与 SVG 编码”提示词，但为了和 slide plan 体系配合，做了收敛。

## 什么时候读这个文件

- 当前页 `render_mode=custom_svg`
- 你要给模型一页一页地产生 SVG 草稿
- 你需要为封面、目录、章节页、结束页写页面提示词

## 硬约束

### 画布

- SVG `viewBox` 必须是 `0 0 1280 720`

### 基本布局

- 采用 Bento Grid 思路，但不是僵硬模板
- 通过卡片大小表达层级
- 保持留白
- 卡片间距至少 20px

### 页面气质

- 高级感
- 简洁感
- 专业感
- 结构优先于装饰

## 页面编排原则

### 封面页

- 大标题必须是第一视觉重心
- 副标题承担解释，不抢主标题
- 主视觉只保留 1 个，不要塞多组图形
- 通常只保留 2-4 个辅助信息点

### 目录页

- 目录只列章节，不展开每页
- 可以用 2 栏或 4 象限排布
- 序号、章节名、短说明分层展示

### 章节页

- 章节编号要醒目
- 一句核心观点比一堆说明更重要
- 章节页是节奏页，不是信息密度页

### 结尾页

- 保留 1 个总结标题
- 3 个以内回顾点
- 如果有 CTA，只保留一个动作

## 页面提示词模板

可直接把下面的模板当作生成 SVG 的基底：

```text
作为精通信息架构与 SVG 编码的专家，你的任务是将完整的文字内容转化为一张高质量、结构化、具备高级感、简洁感和专业感的 SVG 演示文稿页面。

硬约束：
1. SVG viewBox 必须是 0 0 1280 720。
2. 使用 Bento Grid 思维进行信息分组，但布局应由内容驱动，而非死板模板。
3. 所有卡片之间保持至少 20px 间距。
4. 使用卡片尺寸建立层级：最重要的信息必须放进最大卡片。
5. 不要把长段正文塞进狭窄卡片。

请根据以下页面规格输出 SVG 代码：

页面角色：{page_role}
页面标题：{title}
页面副标题：{subtitle}
页面内容：
{content_block}

额外风格要求：
{style_notes}
```

## 和 slide_plan 的衔接方式

### 对 `custom_svg` 页面

从 `slide_plan` 读取：

- `page_role`
- `title`
- `subtitle`
- `content`

再根据角色补一层构图要求。

### 对 `engine` 页面

通常不需要这份提示词，优先走 bundled engine。

## 角色到构图的建议映射

### `cover`

- 主标题居中或左上主导
- 一个高辨识度主视觉
- 可加入年份、主题标签、演讲人信息

### `toc`

- 目录分组清晰
- 建议 2-6 个章节条目
- 可使用序号 + 标题 + 一句解释

### `chapter`

- 大编号
- 章节名
- 一句章节洞察

### `closing`

- 总结标题
- 2-3 条核心回顾
- 可选一个行动建议或结束语

## 语言与密度约束

- 中文页尽量短句化
- 每页只承载一个中心思想
- 不为了“看起来满”而堆内容
- 如果内容过多，优先拆页，不要缩小到读不清

## 常见错误

- 画布比例不是 16:9
- 所有卡片都一样大，失去层级
- 标题和正文视觉差别太小
- 卡片过多，页面像看板而不是演示稿
- 窄卡里放完整段落
