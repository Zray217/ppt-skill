# 汇报演示页面布局（12 种）

每页 slide 归入且仅归入以下 12 种布局之一。选择布局时，同时标记该页的**节奏类型**（hero / non-hero），确保整份 PPT 有呼吸感。

---

## 节奏标记说明

| 节奏 | 含义 | 视觉特征 |
|------|------|---------|
| **hero** | 停顿页、冲击页 | 大字号、大留白、深色背景/色块背景、信息极简 |
| **non-hero** | 信息承载页 | 正常字号、信息密度高、白色/浅色背景 |

**硬规则**：
- 每 3-4 页插入 1 个 hero 页
- 连续 3+ 页 non-hero = 不允许
- 8 页以上 deck 必须有 ≥ 1 个 dark hero + ≥ 1 个 light hero
- 相邻 2 页不允许同一布局

---

## 网格比例速查

汇报场景常用三种两列网格比例：

| 比例 | 英文记法 | 适用 |
|------|---------|------|
| 7:5 | `grid-7-5` | 文字为主 + 辅助图表 |
| 6:6 | `grid-6-6` | 对半对比 |
| 8:4 | `grid-8-4` | 大段文字 + 小数据/图 |

三列和四列网格：

| 比例 | 英文记法 | 适用 |
|------|---------|------|
| 3:3:3 | `grid-3-equal` | 三项并列 |
| 2:4:4 | `grid-2-4-4` | 小标题 + 两列内容 |
| 4:4:4:4 | `grid-4-equal` | 四项能力/步骤 |

## L01 封面（Cover）

**节奏**：hero | **必用**：第 1 页

### 网格

全幅，无网格分割。标题垂直居中或偏左下。

### 布局骨架

```
|                                        |
|                                        |
|     [大标题 48-60pt]                    |
|     [副标题/项目名 20-28pt]             |
|     [日期 · 汇报人 14-16pt]             |
|                                        |
```

### 设计规则

| 规则 | 说明 |
|------|------|
| **背景** | 用 `theme.primary`（深色）或渐变色块制造纵深感 |
| **标题字号** | 48-60pt，Bold，颜色用白/浅色 |
| **副标题** | 20-28pt，比标题小 2 倍以上 |
| **元信息** | 14-16pt，放在标题下方或页面底部，颜色用 `theme.light` |
| **不需要页码 badge** | 封面自成一体 |

### PptxGenJS 示例

```javascript
slide.background = { color: theme.primary };
slide.addText("项目名称", {
  x: 0.8, y: 1.8, w: 8.4, h: 1.5,
  fontSize: 52, fontFace: "Arial", bold: true,
  color: "FFFFFF"
});
slide.addText("副标题 / 项目定位", {
  x: 0.8, y: 3.3, w: 8.4, h: 0.6,
  fontSize: 22, fontFace: "Arial",
  color: theme.accent
});
slide.addText("2026.04 · 汇报人", {
  x: 0.8, y: 4.6, w: 8.4, h: 0.4,
  fontSize: 14, fontFace: "Arial",
  color: "AAAAAA"
});
```

## L03 核心数据（Key Metrics / Big Numbers）

**节奏**：hero 或 non-hero | **用于**：关键指标冲击

### 网格

- 3 个数据 → `grid-3-equal`
- 4 个数据 → `grid-4-equal` 或 2×2
- 6 个数据 → 3×2 grid

### 布局骨架

```
|  核心指标                               |
|  ┌─────────┐ ┌─────────┐ ┌─────────┐  |
|  │  Label   │ │  Label   │ │  Label   │ |
|  │  1,523亿 │ │   27家   │ │  +152%  │ |
|  │  备注    │ │  备注    │ │  备注    │ |
|  └─────────┘ └─────────┘ └─────────┘  |
```

### 设计规则

| 规则 | 说明 |
|------|------|
| **数字字号** | 60-96pt，Bold，用 `theme.accent` 或 `theme.primary` |
| **标签** | 12-14pt，`theme.secondary` |
| **备注** | 10-12pt，muted 色，口径说明放这里 |
| **数字长度** | 建议 2-5 位字符，太长用 K/M/亿 简写 |
| **hero 版本** | 深色背景 + 白字数字，冲击力更强 |
| **non-hero 版本** | 白色背景 + 色块卡片 |
| **数据来源** | **必须标注**，放在备注或卡片底部 |

## L04 要点陈述（Key Message）

**节奏**：non-hero | **用于**：核心论点、结论、定位陈述

### 网格

- 单要点 → 全幅居中
- 2-3 要点 → `grid-3-equal`
- 要点 + 论据 → `grid-7-5`（左文字 右支撑图）

### 布局骨架

```
|  要点标题                               |
|                                        |
|  核心结论文字（18-24pt）                 |
|  支撑说明（14-16pt）                    |
|                                        |
|  [可选：支撑图标/色块]                   |
```

### 设计规则

| 规则 | 说明 |
|------|------|
| **结论文字** | 18-24pt，Bold，一页最多 3 条 |
| **支撑说明** | 14-16pt，每条 1-2 行 |
| **视觉元素** | 每条结论配图标或色块标记，不允许纯文字 |
| **左对齐** | 文字一律左对齐，标题可居中 |

## L05 痛点/问题（Problem Statement）

**节奏**：hero 或 non-hero | **用于**：定义问题、制造紧迫感

### 网格

- 单一痛点 → 全幅居中
- 多痛点 → 垂直列表 + 图标

### 布局骨架

```
|  当前挑战                               |
|                                        |
|  ● 痛点一：问题描述                     |

|                                        |
|  ● 痛点二：问题描述                     |
|    影响 / 数据佐证                      |
|                                        |
|  ● 痛点三：问题描述                     |
|    影响 / 数据佐证                      |
```

### 设计规则

| 规则 | 说明 |
|------|------|
| **痛点数量** | 3 个最佳，不超过 5 个 |
| **hero 版本** | 深色背景 + 大字问题，只放 1 个核心问题 |
| **图标** | 每个痛点配图标（warning/alert/趋势箭头） |
| **数据佐证** | 痛点必须配数字——"效率低"不如"效率仅 35%" |

| 规则 | 说明 |
|------|------|

### 布局骨架

## Bento Grid 卡片组合方法论

**核心理念**：页面不是模板的填空题，而是内容的便当盒。卡片大小由信息重要性决定——最重要的放最大，辅助的放小。对称是安全牌但也是平庸之源。

### 卡片权重体系

| 权重 | 标记 | 视觉面积占比 | 字号范围 | 放哪里 |
|------|------|-------------|---------|--------|
| **核心** | `★` | 40-60% 页面面积 | 28-96pt | 大卡片（主卡片） |
| **支撑** | `◆` | 15-25% 页面面积 | 14-20pt | 中卡片 |
| **辅助** | `●` | 5-15% 页面面积 | 10-13pt | 小卡片/标签/来源 |

### 金字塔原则（页内）

每一页内部也遵循金字塔原理——视觉权重最大的元素 = 该页核心结论：

1. **结论先行**：大卡片放结论/关键数据，不是铺垫
2. **以上统下**：主卡片结论由下方小卡片支撑
3. **归类分组**：同一层级的卡片放在一起
4. **逻辑递进**：从左到右/从上到下 = 从结论到论据

### 卡片尺寸速查（10" × 5.625" 画布）

| 卡片类型 | 宽 | 高 | 适合内容 |
|---------|------|------|---------|
| 全幅大卡 | 8.8" | 3.5" | 核心结论、大数字 |
| 左侧大卡 | 5.5" | 3.5" | 文字/方案说明 |
| 右侧窄卡 | 3.0" | 3.5" | 数据/图表/指标 |
| 顶部横卡 | 8.8" | 1.5" | 关键数据行 |
| 底部小卡 | 2.7" | 1.5" | 分项说明 |
| 标签卡 | 1.8" | 1.0" | 图标+短文字 |
| 角卡 | 2.0" | 1.2" | 小指标/标签 |

**卡片间距**：所有卡片之间统一 `gap = 0.15"`（约 15px），不要忽大忽小。

## L13 Hero+Sub（核心结论 + 支撑）

**节奏**：non-hero | **用于**：核心论点 + 2-3 个支撑论据

### 网格

1 大卡(60%面积) + 2-3 小卡(40%面积)。左右或上下排布。

### 布局骨架

```
|  ┌──────────────────┐  ┌─────────┐  |
|  │                  │  │ 支撑一  │  |
|  │   ★ 核心结论     │  │ ◆       │  |
|  │   大字/大数字    │  ├─────────┤  |
|  │                  │  │ 支撑二  │  |
|  │                  │  │ ◆       │  |
|  └──────────────────┘  └─────────┘  |
```

### 设计规则

| 规则 | 说明 |
|------|------|
| **大卡字号** | 28-48pt，核心结论或关键数字 |
| **小卡字号** | 14-16pt，支撑论据 |
| **大卡背景** | 可用 `theme.light` 或深色色块制造对比 |
| **小卡数量** | 2-3 个，不超过 4 个 |
| **字号差** | 大卡字号 ≥ 小卡字号 × 2 |
| **金字塔结构** | 大卡 = 结论，小卡 = 论据——先看到结论 |

### PptxGenJS 示例

```javascript
// 主卡片
slide.addShape(pres.shapes.RECTANGLE, {
  x: 0.6, y: 1.3, w: 5.5, h: 3.5,
  fill: { color: theme.light }
});
slide.addText("核心结论", {
  x: 0.9, y: 1.6, w: 5.0, h: 1.0,
  fontSize: 36, fontFace: "Arial", bold: true,
  color: theme.primary
});
slide.addText("支撑说明文字", {
  x: 0.9, y: 2.8, w: 5.0, h: 1.5,
  fontSize: 15, fontFace: "Arial",
  color: theme.secondary
});

// 右侧小卡片 × 2
const subCards = [
  { y: 1.3, title: "支撑一", desc: "数据/论据" },
  { y: 3.0, title: "支撑二", desc: "数据/论据" }
];
subCards.forEach(c => {
  slide.addShape(pres.shapes.RECTANGLE, {
    x: 6.3, y: c.y, w: 3.2, h: 1.5,
    fill: { color: theme.light }
  });
  slide.addText(c.title, {
    x: 6.5, y: c.y + 0.2, w: 2.8, h: 0.4,
    fontSize: 16, fontFace: "Arial", bold: true,
    color: theme.primary
  });
  slide.addText(c.desc, {
    x: 6.5, y: c.y + 0.7, w: 2.8, h: 0.5,
    fontSize: 13, fontFace: "Arial",
    color: theme.secondary
  });
});
```

## L14 非对称双栏（主 + 辅）

**节奏**：non-hero | **用于**：文字为主 + 数据提亮，或方案 + 侧边指标

### 网格

2/3 左栏（主内容）+ 1/3 右栏（辅助信息/数据/图表）

### 布局骨架

## L16 混合网格（自由组合）

**节奏**：non-hero | **用于**：多维度信息、仪表盘、综合展示——最适合 Bento 精神

### 网格

自由组合不同尺寸的卡片，没有固定比例。由内容权重决定每张卡的大小。

### 布局骨架（示例）

```
|  ┌──────────────┐  ┌──────┐  ┌──────┐  |
|  │  ★           │  │ ◆    │  │ ◆    │  |
|  │  大卡(核心)   │  │      │  │      │  |
|  │              │  ├──────┤  ├──────┤  |
|  │              │  │ ●    │  │ ●    │  |
|  └──────────────┘  └──────┘  └──────┘  |
```

### 设计规则

| 规则 | 说明 |
|------|------|
| **卡片数量** | 4-7 张，不超过 8 张 |
| **尺寸变化** | 至少 3 种不同尺寸——全一样 = 失去层级 |
| **最大卡:最小卡** | 面积比 ≥ 3:1 |
| **对齐** | 卡片边缘必须对齐（左对齐或网格对齐），不要"漂浮" |
| **留白** | 卡片之间 0.15"，不要挤在一起 |
| **来源标注** | 每张数据卡的底部放来源，10pt muted 色 |

### PptxGenJS 坐标计算示例

```javascript
// 混合网格坐标规划（10" × 5.625" 画布）
const margin = 0.6;    // 页边距
const gap = 0.15;      // 卡片间距
const contentW = 10 - margin * 2;  // 8.8"
const contentH = 5.625 - margin - 1.2; // 3.825" (留出标题区)

// 大卡片：占据 60% 宽度
const bigW = contentW * 0.58;     // ≈ 5.1"
const bigH = contentH;            // ≈ 3.825"

// 右侧：两个中卡片叠放
const medW = contentW - bigW - gap;  // ≈ 3.55"
const medH = (contentH - gap) / 2;   // ≈ 1.84"

const bigX = margin;                    // 0.6
const medX = margin + bigW + gap;       // 5.85
const topY = 1.4;                       // 标题下方
const medTopY = topY;
const medBotY = topY + medH + gap;
```
