# 质量检查清单 — P0/P1/P2/P3 分级

生成完必须逐项对照。**P0 级问题必须全过，否则不允许交付。**

---

## P0 · 一定不能犯的错

### 节奏与结构

- [ ] **hero 与 non-hero 交替**：连续 3+ 页同节奏 = 失败
- [ ] **8+ 页 deck 必须有 ≥ 1 dark hero + ≥ 1 light hero**
- [ ] **不能只有 light 正文页**，必须有 dark 正文页
- [ ] **相邻 2 页不能同一布局**
- [ ] **同一布局整个 deck 不超过 3 次**
- [ ] **12+ 页 deck 至少使用 6 种不同布局**
- [ ] **10+ 页 deck 至少 3 页使用非对称组合**（L13/L14/L15/L16）
- [ ] **对称等分布局不超过 3 页**——对称是安全牌也是平庸之源

### Bento 卡片组合

- [ ] **卡片大小由内容权重驱动**——核心信息(★) 占最大卡片，辅助(●) 占最小
- [ ] **最大卡片面积 ≥ 最小卡片 × 3**——面积差不足 = 没有层级
- [ ] **大卡片字号 ≥ 小卡片字号 × 2**——没有字号差就没有权重差
- [ ] **每页至少 2 种卡片尺寸**——全等 = 失去层级感
- [ ] **卡片间距统一 0.15"**——忽大忽小 = 杂乱
- [ ] **页内金字塔**：视觉权重最大的元素 = 该页核心结论，不是铺垫

### 内容与信息

- [ ] **总结页必须有明确行动项**——"谢谢"不是终点
- [ ] **数据必须标注来源**——没有来源的数据不如不放
- [ ] **核心结论必须在开场 3 页内出现**——决策者的注意力窗口有限
- [ ] **一页一核心信息**——一页承载多个无关论点 = 失败
- [ ] **痛点必须配数据佐证**——"效率低"不如"效率仅 35%"

### 视觉基础

- [ ] **大标题字号 ≥ 36pt**——不够突出 = 失败
- [ ] **标题与正文字号差 ≥ 2 倍**
- [ ] **body text 左对齐**——居中正文 = 失败
- [ ] **每页至少一个非文字视觉元素**——纯文字页 = 失败
- [ ] **不用 emoji 作图标**
- [ ] **不用装饰线条**（标题下划线、分隔线等 AI 味重的元素）
- [ ] **不重复同一布局**——相邻页视觉不能一样

### 技术约束

- [ ] **hex 颜色不带 #**——`"FF0000"` 正确，`"#FF0000"` 损坏文件
- [ ] **不把 opacity 编码进 hex**——用 `opacity` 属性
- [ ] **createSlide() 不用 async/await**——compile.js 不会 await
- [ ] **不跨调用复用 option 对象**——PptxGenJS 会原地修改
- [ ] **封面页不加页码 badge**
- [ ] **其他所有页必须有页码 badge**

---

## P1 · 排版节奏

### hero 页设计

- [ ] **hero 页信息极简**——最多 3 个视觉元素
- [ ] **hero 页留白 ≥ 40%**
- [ ] **hero 页大字号**——核心文字 48pt+
- [ ] **章节幕封交替 dark/light**——两个连续幕封不能同色

### non-hero 页设计

- [ ] **non-hero 页信息密度合理**——5-10 个视觉元素
- [ ] **卡片/容器用 `theme.light` 做背景色**
- [ ] **数据数字 60-96pt**，比周围文字大 3 倍以上

### 跨页一致性

- [ ] **同一 deck 的间距系统一致**——要么 0.3" 要么 0.5"，不要混用
- [ ] **同一 deck 的圆角风格一致**——选了 Soft 就不要混 Rounded
- [ ] **术语全 deck 统一**——同一个词 1 个写法
- [ ] **页码 badge 位置统一**——x: 9.3", y: 5.1"

---

## P2 · 视觉打磨

### 配色

- [ ] **accent 色不超过页面面积 10%**——太多 = 失去强调效果
- [ ] **hero 页背景用 `theme.primary`**——正文页用 `theme.bg`**
- [ ] **Before/After 对比**：旧侧 muted + opacity 0.5，新侧 accent 高亮
- [ ] **能力矩阵**：优势项用 accent 标记

### 字体

- [ ] **正文不用 Bold**——只有标题用 Bold
- [ ] **数据来源/备注用 10-12pt muted 色**
- [ ] **中文大标题 ≤ 8 字**——超长用 `<br>` 断行
- [ ] **一页最多 3 种字号**

### 布局

- [ ] **两列网格**：7:5（文字为主）、6:6（对半）、8:4（大段文字+小图）
- [ ] **三列网格**：3:3:3（并列）
- [ ] **图片用标准比例**（16:10/4:3/3:2/1:1/16:9）
- [ ] **图片路径用相对路径**——`imgs/xxx.png`

---

## P3 · 操作细节

- [ ] **compile.js 的 theme 对象**使用 5 个标准 key（primary/secondary/accent/light/bg）
- [ ] **slideConfig 包含 type + rhythm + index + title**
- [ ] **文件命名**：`slide-01.js`、`slide-02.js`...
- [ ] **图片命名**：`{页号}-{语义}.{ext}`
- [ ] **最终输出**：`slides/output/presentation.pptx`
- [ ] **markitdown 验证**：提取文字确认内容完整、无占位符残留
- [ ] **grep 检查占位符**：`python -m markitdown output.pptx | grep -iE "xxxx|lorem|ipsum|placeholder"`

---

## 自检流程

1. 生成 slides → Extract text with `python -m markitdown output.pptx` → Review content
2. **对照 P0 清单逐项检查**——一项不过就不交付
3. 修复问题
4. **重新验证受影响的 slides**——一个修复往往制造另一个问题
5. 重复直到完整检查无新问题

**Do not declare success until you've completed at least one fix-and-verify cycle.**
